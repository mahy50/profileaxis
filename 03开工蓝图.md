# AI 驱动的 3D 铝型材网页设计系统首版开工蓝图

## 开工结论

现在直接开工，不再继续发散：首版冻结为**受约束的会话式装配设计器**，真相源统一为 `ProjectDocument / resolvedDsl`，前端负责本地建模、规则重算、局部编辑与预览，后端只负责 AI 编排、catalog 版本分发与 ETL、正式 PDF 导出。今天就按 `packages/domain → packages/schemas → packages/stdlib → packages/rules → packages/modeler → apps/web 壳层 → apps/api` 这个顺序启动；在 `draftDsl -> resolvedDsl -> render` 打通之前，不做 UI 打磨，不接商城 SKU，不扩结构类型。fileciteturn0file0fileciteturn0file1

## MVP 架构冻结版

**首版最终边界**

首版边界按下表冻结，超出范围一律后置。fileciteturn0file0fileciteturn0file1

| 分类 | 内容 | 冻结说明 |
|---|---|---|
| 必须做 | 自然语言需求解析 | 打通 `intentDsl -> confirmationDsl -> draftDsl`。 |
| 必须做 | 确认层 | 只确认尺寸、层数、承重档位、是否靠墙、是否允许后撑、型材系列偏好等关键决策。 |
| 必须做 | 规则合法化 | `draftDsl` 不能直接落成结构，必须经过规则层得到 `resolvedDsl`。 |
| 必须做 | 前端本地参数化建模 | 主要编辑动作不依赖后端回包。 |
| 必须做 | 3D 选中与结构树同步 | Scene 点击、树选中、属性面板联动。 |
| 必须做 | 局部自然语言编辑 | 首版支持 6~8 个语义动作，且必须走 `ReferenceContext -> EditIntent -> Command`。 |
| 必须做 | 检测模块 | 输出 blocker / warning / info，支持定位与修复入口。 |
| 必须做 | 设计 BOM | 从 `resolvedDsl` 生成，规格级，不带实时价格。 |
| 必须做 | 交易 BOM | 单供应商、单地区、固定 `catalogVersion`。 |
| 必须做 | 三视图尺寸视图 | 前 / 右 / 顶，基于 `resolvedDsl` 正投影生成 SVG。 |
| 必须做 | PDF 方案单 | 后端固定模板导出。 |
| 必须做 | 本地临时保存 | IndexedDB 存项目、命令、快照。 |
| 必须做 | 撤销 / 重做 | 语义命令模式。 |
| 必须做 | 版本快照 | 手动快照 + 自动快照。 |
| 可以降级实现 | 尺寸标注 | 只做工程可用的链式标注，不做 CAD 级标注编辑器。 |
| 可以降级实现 | 连接件 3D | 允许简化几何体，不影响 BOM 正确性。 |
| 可以降级实现 | 局部编辑歧义处理 | 一旦歧义，强制追问，不追求“一句到位”。 |
| 可以降级实现 | 属性替换 | 只做按角色或按层的批量替换，不做复杂条件替换器。 |
| 可以降级实现 | 检测解释 | 首轮可模板化文案，AI 解释可以后接。 |
| 明确不做 | 自由手工从零建模 | 不提供 Web CAD 式自由拉杆、拉线、任意 joint 编辑。 |
| 明确不做 | 非矩形 / L 型 / 斜撑混合空间框架 | 二期再做。 |
| 明确不做 | 复杂仿真 | 不做强度、稳定性、碰撞仿真。 |
| 明确不做 | 实时价格 / 库存 | 只认静态 catalog 版本。 |
| 明确不做 | 账号、云同步、多人协作 | 首版全部后置。 |
| 明确不做 | 直接下单 | 最多导出交易 BOM，不接订单链路。 |
| 明确不做 | 多供应商映射 | 首版只允许单供应商。 |
| 明确不做 | 门、抽屉、面板生态扩展 | 仅预留接口，不做交互。 |

**最终系统架构**

推荐架构只保留一套主方案，不再保留并行路线。fileciteturn0file0fileciteturn0file1

| 模块 | 职责 | 依赖方向 | 是否必须 shared | 首版放置位置 |
|---|---|---|---|---|
| 前端工作区 | 聊天输入、确认层、结构树、属性面板、检测面板、BOM 面板、命令入口 | 依赖 `domain/schemas/rules/bom/modeler/render-babylon/export` | 否 | `apps/web` |
| 3D 渲染层 | `SceneViewModel` 渲染、选中、高亮、hover、drag preview、mesh diff 更新 | 只依赖 `modeler` 输出；不反写业务真相 | `modeler` 必须 shared，`render-babylon` 不必 shared | 前端 |
| ProjectDocument / DSL | 项目真相、DSL 存档、版本信息、命令游标、快照索引 | 位于依赖底座；被所有业务模块消费 | 是 | `packages/domain` + `packages/schemas` |
| 规则引擎 | `draftDsl -> resolvedDsl`，默认值补全、合法化、规格解析、确定性求解 | 依赖 `domain + stdlib + schemas` | 是 | 前端 Worker 主执行；后端可复算 |
| 检测模块 | 对 `resolvedDsl` 做 blocker / warning / info 检测，并返回修复提示 | 依赖 `rules + stdlib`，不依赖渲染层 | 是 | 前端 Worker 主执行；导出前后端可复验 |
| BOM 模块 | 设计 BOM 聚合、交易 BOM 映射、pack rounding、stock bar 采购量 | 依赖 `domain + stdlib`，输入是 `resolvedDsl` 或 `DesignBomItem[]` | 是 | 前端做预览；后端持有映射真相 |
| AI Orchestrator | 需求解析、草案生成、局部编辑意图、检测解释；统一 strict schema | 依赖 `ai-contracts + schemas + capabilityMatrix summary` | 合同包必须 shared | 后端 `apps/api` |
| 标准库 / SKU 映射 | profile、connector、compatibility、SKU map、supplier policy、catalog version | `stdlib` 为共享底座，ETL 与 catalog 发布在后端 | `stdlib` 必须 shared | `packages/stdlib` + `tools/catalog-etl` + `apps/api` |
| 导出模块 | 三视图 SVG、导出 payload、PDF 模板渲染 | 依赖 `domain + bom`；不依赖 Scene 截图为真相 | `export` 必须 shared | SVG 前端生成，PDF 后端生成 |

依赖方向固定为：`domain / schemas -> stdlib -> rules -> modeler -> render-babylon`，并行派生 `resolvedDsl -> checks`、`resolvedDsl -> designBom -> tradeBom`、`resolvedDsl + BOM + SVG -> export`。AI 只产出 `intentDsl / draftDsl / EditIntent / check explanation`，不允许直接产出 `resolvedDsl`、BOM、检测结论或 SKU。fileciteturn0file0fileciteturn0file1

前后端边界也固定如下：

| 能力 | 前端 | 后端 |
|---|---|---|
| 项目真相持有 | 是 | 否 |
| 规则主执行 | 是 | 可复算 |
| 3D 渲染与交互 | 是 | 否 |
| 局部编辑解析后的命令执行 | 是 | 否 |
| 设计 BOM 预览 | 是 | 可复算 |
| 交易 BOM 预览 | 是，基于下载的 catalog cache | 导出时 authoritative mapping |
| AI 编排 | 否 | 是 |
| catalog ETL / 版本发布 | 否 | 是 |
| PDF 正式导出 | 否 | 是 |

**首版关键技术决策冻结**

所有关键技术决策按下表一次冻结。fileciteturn0file0fileciteturn0file1

| 决策项 | 结论 | 原因 | 风险 | 首版落地建议 |
|---|---|---|---|---|
| Babylon.js | 采用 | 首版是受约束编辑器，不是纯展示器；Babylon 原生具备 utility layer、gizmo、selection outline、thin-instance picking、asset container。 | 团队若缺 Babylon 经验，前两周需要 spike。 | 只建 `packages/render-babylon`，不做 Babylon/Three 双实现。唯一备选是二期再抽 `render-three`。 |
| Web Worker | 采用 | 规则重算、BOM、检测、本地大对象序列化不应阻塞 UI。 | structured clone 成本与调试复杂度上升。 | Worker 仅承载 `rules/checks/bom/serialization`；渲染仍留主线程。 |
| 命令模式 | 采用 | 撤销/重做、快照恢复、局部编辑、自动修复都依赖语义命令。 | `payload` 容易失控，命令粒度可能漂移。 | 只允许语义命令，例如 `resizeBay / moveLevel / toggleBrace`，禁止几何命令。 |
| 状态机 | 采用，但只管工作流 | `输入 -> 确认 -> 生成 -> 检测 -> 导出` 是明确流程，适合状态机；不适合管理整个场景。 | 若把场景和文档都放进状态机，会严重过重。 | 仅在工作流层使用，范围限于聊天/确认/导出流程。 |
| IndexedDB | 采用 | 项目文档、命令、快照、导出产物都属于结构化大对象，本地优先必须落地。 | schema 升级和迁移要早做。 | 设计 `projects / snapshots / commands / exports / catalogCache` 五类 object store；`localStorage` 只留最近项目和 UI 偏好。 |
| Monorepo | 采用 | schema、规则、标准库、BOM、建模、导出合同必须前后端共享，单仓同步成本最低。 | 早期拆包纪律要求高。 | 直接按 `apps + packages + tools` 建仓，不走“先单体后拆包”。 |
| shared schemas / rules package | 采用 | 不允许前后端各自实现一套“差不多”的 contract 与规则，否则 BOM、检测、导出会漂。 | 版本漂移会直接破坏历史项目复现。 | `packages/schemas` 与 `packages/rules` 作为底座包，任何应用层禁止反向依赖。 |
| PDF 导出 | 后端 | 字体、分页、表格换行、版式稳定都需要统一环境。 | 服务端模板与前端预览可能出现偏差。 | 前端只提交 `ProjectDocument + BOM + SVG + preview`；后端固定模板渲染。若失败，唯一降级是先导出 `SVG + CSV + JSON bundle`。 |

**首版最小闭环**

一句话定义：**用户输入一句在支持范围内的需求，完成必要确认后生成一个可编辑的矩形储物架；随后完成一次局部编辑，系统给出无 blocker 的 `resolvedDsl`、设计 BOM、交易 BOM 和 PDF，即视为首版可上线验证。** fileciteturn0file0fileciteturn0file1

| 闭环能力 | 通过条件 | 未通过就不算 MVP |
|---|---|---|
| 需求解析与确认 | 能稳定产出 `intentDsl` 与 `confirmationDsl`，关键项未确认时能阻断生成 | AI 直接跳过确认层 |
| 结构合法化 | `draftDsl` 必须能变成合法 `resolvedDsl` | 只有草案，没有可解算结构 |
| 可编辑工作区 | 3D、结构树、属性面板、选中同步可用 | 只能看，不能改 |
| 局部编辑与历史 | 至少支持整体尺寸、层增减、层高、后撑、型材系列、横梁增删等核心动作，且撤销/重做可用 | 编辑不能回滚，或编辑直接改几何 |
| BOM 与导出 | 设计 BOM、交易 BOM、三视图 SVG、PDF 都能从当前 `resolvedDsl` 派生 | 渲染有了，但交付链路不通 |

| PDF 与尺寸视图一致性 | SVG、PDF、当前模型三者容易不一致 | 用固定 fixture 比对 `resolvedDsl` 尺寸、三视图 SVG、PDF 关键字段 | 前 / 右 / 顶关键尺寸与 BOM 摘要一致 | 暂时导出 `SVG + CSV + JSON bundle`，推迟 PDF |
| AI strict schema 稳定性 | schema 合法但语义错误，或 schema 漂移导致接口崩溃 | 对每个 AI endpoint 做 contract test，覆盖 refusal / schema_error | 所有 AI 响应只能落在 `ok / refusal / schema_error` 三态内 | 先缩成模板优先，AI 只做 intent 解析，不生成 draft |
